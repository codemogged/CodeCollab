# 05 — Build, Deploy & Configuration

This doc is the reference for every script, config file, installer, and batch helper that turns the
source tree into a shippable Windows desktop app and puts it on an end-user machine.

---

## 5.1 `package.json` scripts

| Script | Command | When to use |
|---|---|---|
| `dev` | `next dev --turbopack` | Browser-only iteration (no Electron, no IPC). |
| `dev:electron` | `concurrently "next dev --turbopack --port 3000" "wait-on http://localhost:3000 && electron ."` | Full local dev loop with live IPC. |
| `build` | `next build` | Next.js static export → `./out/`. |
| `build:electron` | `next build && electron-builder` | Package the app → `./dist-electron/`. |
| `deploy` | `next build && electron-builder && powershell -NoProfile -ExecutionPolicy Bypass -File scripts/deploy-install.ps1` | Full release + deploy to the local install folder. |
| `build:icon` | `powershell … scripts/generate-icon.ps1` | Regenerate `build/icon.ico` / `icon.png`. |
| `postinstall` | `electron-builder install-app-deps` | Rebuild native deps for Electron after `npm install`. |
| `start` | `next start` | Serve built Next output without Electron (rarely used). |
| `lint` | `eslint .` | Lint the whole repo. |

## 5.2 electron-builder config (inside `package.json`)

- `appId: "com.codebuddy.app"`
- `productName: "CodeBuddy"`
- `directories.output: "dist-electron"`
- `directories.buildResources: "build"`
- `files: ["electron/**/*", "out/**/*"]`
- `extraResources: [{ from: "build/icon.png", to: "icon.png" }]`

**Windows (NSIS) target:**

- `oneClick: false`, `perMachine: false`
- `allowToChangeInstallationDirectory: true`
- Creates Desktop + Start Menu shortcuts
- Uses `build/icon.ico` for installer header / installer / uninstaller icons
- `deleteAppDataOnUninstall: false` (user data survives uninstall)
- `runAfterFinish: true`

**macOS target:** DMG, category `public.app-category.developer-tools`.
**Linux target:** AppImage, category `Development`.

Build output lives in `dist-electron/`:

- `CodeBuddy Setup 0.2.0.exe` — NSIS installer.
- `CodeBuddy Setup 0.2.0.exe.blockmap` — delta-update metadata.
- `latest.yml` — electron-updater manifest.
- `win-unpacked/` — unpacked portable directory (`CodeBuddy.exe`, DLLs, `resources/app/`, locales).

## 5.3 Root config files

| File | Key settings |
|---|---|
| `next.config.ts` | `output: "export"` — static site generation so Electron can load the bundle directly. |
| `tsconfig.json` | `target: ES2017`, strict mode, `@/*` path alias → `src/`. |
| `eslint.config.mjs` | Extends `next/core-web-vitals` + `next/typescript`. |
| `postcss.config.mjs` | Autoprefixer + Tailwind. |
| `tailwind.config.ts` | Dark mode via `class`; custom surface / text / accent tokens that mirror the CSS variables defined in `src/app/globals.css`. |

## 5.4 Root batch / PowerShell helpers

| File | What it does |
|---|---|
| `debug-start.bat` | Kills any running CodeBuddy, checks that `dist-electron/win-unpacked/CodeBuddy.exe` exists, launches it with verbose logging. Developer-facing. |
| `FRESH-START.bat` | Kills CodeBuddy, wipes `%APPDATA%\codebuddy`, re-deploys latest build, relaunches with clean onboarding. Used when testing first-run flows. |
| `UNINSTALL-ALL.bat` | Delegates to `UNINSTALL-ALL.ps1` with auto-elevation. |
| `UNINSTALL-ALL.ps1` | **Elevates to admin**, removes CodeBuddy + settings, Codex CLI, Claude Code, GitHub Copilot CLI, GitHub CLI, Node.js, Python, Git. "Nuke it all" path. |
| `UPDATE.ps1` | Right-click in the install folder to update in place: closes the app, downloads new artifacts, copies over them while preserving `%APPDATA%\CodeBuddy`. |
| `deploy-clean.ps1` | Full clean redeploy: kill processes → delete old install and caches → copy fresh build → register auto-update. |

---

## 5.5 `scripts/` helpers

| File | Purpose |
|---|---|
| `scripts/deploy-install.ps1` | Called by `npm run deploy`. Stops any running CodeBuddy, clears the default install folder (`Desktop\CodeBuddy Install`), copies `dist-electron/win-unpacked/*` in, and adds `FRESH-START.bat` / `UPDATE.ps1` / `UNINSTALL-ALL.*` alongside. |
| `scripts/generate-icon.ps1` | Generates `build/icon.ico` in multiple sizes (256/128/64/48/32/16) via `.NET System.Drawing` with a gradient and the letters "CB". |
| `scripts/update-desktop-shortcut.js` | Creates / updates a Desktop shortcut to `dist-electron/win-unpacked/CodeBuddy.exe` using PowerShell's COM `WScript.Shell`. |

---

## 5.6 `build/` and `dist-electron/`

`build/` contains `icon.ico` (Windows) and `icon.png` (macOS / Linux). Everything else in `dist-electron/`
is regenerated by `electron-builder` — it is **not** hand-edited and is typically git-ignored in
release branches.

## 5.9 End-to-end release flow

```
                ┌────────────────────────────────────────┐
                │        DEV                             │
                │                                        │
                │ npm run dev:electron                   │
                │  ├─ Next dev server :3000              │
                │  └─ Electron pointing at :3000         │
                └─────────────────┬──────────────────────┘
                                  │
                ┌────────────────────────────────────────┐
                │        RELEASE BUILD                   │
                │                                        │
                │ npm run build:electron                 │
                │  ├─ next build             → out/      │
                │  └─ electron-builder       → dist-     │
                │                              electron/ │
                └─────────────────┬──────────────────────┘
                                  │
                ┌────────────────────────────────────────┐
                │        DEPLOY TO LOCAL INSTALL         │
                │                                        │
                │ npm run deploy                         │
                │  └─ scripts/deploy-install.ps1         │
                │      copies win-unpacked/* → Desktop\  │
                │      CodeBuddy Install + helper .bats  │
                └─────────────────┬──────────────────────┘
                                  │
                ┌────────────────────────────────────────┐
                │        DISTRIBUTE                      │
                │                                        │
                │  Setup 0.2.0.exe  (NSIS installer)     │
                │  0.2.0.exe        (portable)           │
                │  latest.yml       (electron-updater)   │
                └────────────────────────────────────────┘
```

End-user flow:

1. Run `CodeBuddy Setup 0.2.0.exe` → NSIS installs to `%LOCALAPPDATA%\Programs\CodeBuddy`, creates a Desktop shortcut, and launches the app.
2. First launch → `/onboarding` (installs Git, Node, Python, GitHub CLI, Copilot CLI, Claude Code, Codex CLI as needed).
3. Updates → right-click `UPDATE.ps1` inside the install folder. Preserves `%APPDATA%\CodeBuddy`.
4. Full removal → right-click `UNINSTALL-ALL.ps1` for a nuclear option (removes every dev tool CodeBuddy installed).
