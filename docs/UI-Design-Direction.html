Foundation
Design Principles
Every concept in this document follows these four constraints. When options conflict, these principles break the tie.
01
Calm over clever
Reduce visual noise. Every element earns its pixels. Whitespace is a feature. The UI should feel like a deep breath — no anxiety, no overwhelm, just the next thing to do.
Reference: Apple Human Interface Guidelines — "Deference"
02
Show the path, hide the plumbing
Git commands, P2P connections, model routing — all invisible. The user sees "your friend joined" and "code synced." Never show infrastructure. GitHub is a power user menu, not a daily surface.
Reference: Apple iCloud sync — you never see it work
03
One action per moment
Each screen has a single primary action. Modals have one button that matters. Settings don't require decisions. The user should always know what to do next without reading.
Reference: Anthropic Claude — single input, single output
04
Friends are present, not features
Collaboration isn't a panel to open. Friend presence is ambient — avatar rings glowing, typing indicators, cursors in chat. It should feel like being in the same room, not using a collaboration tool.
Reference: Figma multiplayer — cursors as presence

Page 04
PM Chat
The AI project manager conversation. Currently: full chat + model selector + file attachments + P2P peer streams. Goal: make this feel like the natural way you talk to an AI about building something, not a dev tool panel.
Claude-Style Clean Thread
Strip the chat to its essence: messages and a composer. Model selection moves to a subtle pill in the composer bar (not a dropdown panel). File attachments are drag-and-drop with a ghost preview. Friend messages interleave naturally with your messages, distinguished only by avatar and subtle color tint. The thread reads like a conversation, not a tool.

Files & Version Control
The git-powered file browser. Currently: directory listing + Monaco editor + git status tabs. Goal: make git feel invisible while keeping its power accessible. Users should never feel like they're "doing git."
iCloud-Style File Manager
Present files like Apple's Finder or iCloud Drive — icon grid or list view with thumbnail previews for images, syntax-colored previews for code files. Sync status is shown as a subtle cloud icon per file (synced / syncing / local only). Git operations happen behind the scenes; the user sees "Synced 2 min ago" at the top, not "3 commits ahead."

Timeline View
Instead of showing current files, show the project as a timeline of changes. Each entry: who changed what, when, and why (AI-generated commit summaries). Clicking an entry shows the diff. This makes the file page about understanding the project's history, not just browsing files. Files are still accessible via a search bar at the top.

Settings
Global user settings. Currently: long scrolling form. Goal: organized, calm, nothing breaks. Inspired by macOS System Settings (grouped panes) and Anthropic's settings (clean rows).
macOS System Settings
Left sidebar with category groups (Account, Appearance, AI Providers, Advanced). Right pane shows the selected category's settings. Each setting is a clean row: label left, control right. Toggles, dropdowns, and text inputs are the only controls. No save button — everything auto-saves with a subtle confirmation animation.

Preview & App Runner
Run and preview the app. Currently: webview with device presets + terminal output. Goal: feel like using a real browser, with the build process abstracted behind a single button.
Safari-Style Preview
A minimal browser chrome with URL bar (read-only, showing localhost), device frame selector as subtle icons (desktop/tablet/phone), and a "Run" button that pulses while building. Navigation Sidebar
The persistent project sidebar. Currently: 220px expanded rail with text labels and chat tree. Goal: reduce footprint, increase scanability. The sidebar should help you navigate, not demand attention.
Icon Rail with Flyout
Default state: 56px icon-only rail (like Slack, Figma, VS Code). Hovering an icon shows a tooltip. Clicking a nav icon navigates immediately. The chat tree (subprojects → tasks → threads) lives in a flyout panel that appears when you hover or click the "PM Chat" icon. This gives maximum screen real estate to content while keeping navigation one click away.

Reference
Recommended Design Tokens
Refined from the current system. Subtle shifts toward a more polished, Apple-grade palette while keeping the warm personality.
Token Current Recommended Rationale
Corner radius (cards) 1.2rem (19px) 16px Slightly tighter. Apple uses 12–16px for cards. Current is slightly balloon-like at small sizes.
Corner radius (buttons) 1.85rem (30px) 12px (standard) / 100px (pills) Two tiers: standard rounded for primary actions, full pill for tab switchers and tags.
Shadow (cards) 0 1px 4px rgba(0,0,0,0.04) 0 1px 3px rgba(0,0,0,0.04), 0 4px 12px rgba(0,0,0,0.02) Layered shadow. More depth without heaviness. Apple's approach.
Surface blur backdrop-blur(16px) backdrop-blur(24px) Stronger frosted glass. More clearly distinguishes floating surfaces from background.
Accent: Orange #ff9f1c #f59e0b (Amber 500) Slightly warmer, better contrast on both light and dark. Tailwind amber.
Accent: Purple #7c5cfc #7c3aed (Violet 600) Deeper purple. More authority. Better dark mode contrast.
Body font size Mixed (13–15px) 14px base, 12px secondary Consistent two-tier type scale. Fewer font-size decisions across components.
Spacing scale Tailwind defaults (mixed) 4 · 8 · 12 · 16 · 24 · 32 · 48 Constrained 7-step scale. Creates rhythm. Eliminates arbitrary gap values.
Sidebar width 220px expanded 56px (collapsed) / 220px (flyout) Icon rail by default. More space for content. Fly-out for deep nav.
Animation curve ease (CSS default) cubic-bezier(0.16, 1, 0.3, 1) Apple's spring-like ease-out. Snappy start, gentle landing. Every transition feels intentional.
