The Right-Side Navbar

The current navbar sits at the top. In the Monolith redesign, navigation moves to a right-side vertical rail — an unconventional but deliberate choice that distinguishes CodeBuddy from every other tool and keeps the left side clear for content flow.

Why Right Side?

Every IDE, every app, every tool puts navigation on the left. By placing it on the right, CodeBuddy does something nobody else does — your eye enters the app from the left (where content begins) and the nav is a quiet, peripheral companion on the right. It's like a table of contents in a book: you know it's there, you glance at it when needed, but it never interrupts your reading.

The right rail also means: no fighting with the OS window controls (top-left on Mac, top-right on Windows — our rail sits below those). And when the IDE clone tab is open, the right rail becomes the bridge between "CodeBuddy mode" and "VS Code mode."

Right Rail — Dimensions

  • Width: 52px collapsed (icon-only), 200px expanded (hover/pin)
  • Height: 100vh
  • Position: fixed right-0, z-50
  • Background: void (#08080a) — lives outside the monolith panel
  • Border-left: 1px solid edge
  • Padding: 0.5rem 0
  • Expand trigger: hover (200ms delay) or pin toggle (⌘\)
  • Expand animation: width 200ms ease-out

Right Rail — Behavior

  • Collapsed = icon glyph only (18×18px)
  • Expanded = icon + label + optional badge
  • Hover on rail: expands after 200ms dwell
  • Leave rail: collapses after 300ms
  • Pin mode: stays expanded, ⌘\ toggles
  • Active item: bg highlight + accent border-right 2px
  • Hover item: bg stage-up, 150ms
  • Click: navigate + collapse (unless pinned)

/project/ide — VS Code IDE Clone

The IDE Tab — VS Code, Recreated

A pixel-perfect recreation of VS Code's editing experience with GitHub Copilot integrated — built directly into CodeBuddy as a full tab. Users can prompt, edit code, browse files, and see Copilot responses all within the same app they use to manage their projects.

Why: The Freestyle tab gives you chat + a small editor. But developers want the real thing — the full VS Code experience where you can browse your entire codebase, have multiple files open, see the file tree, use the terminal, AND chat with an AI in a side panel. This tab delivers exactly that.

How: We render a full VS Code-like IDE layout inside the monolith. In Full layout mode (100% width minus the right rail), the panel becomes VS Code. Activity Bar on the left. Sidebar. Editor group. Copilot chat panel on the right. The only difference from actual VS Code: our right rail replaces VS Code's right edge, and our design language unifies the IDE with the rest of CodeBuddy.

The result: Users never leave CodeBuddy to code. They go from PM Chat (plan what to build) → IDE (build it) → Preview (see it) → all within the same app.
Layout mode: Full — The monolith panel expands to fill the entire viewport minus the 52px right rail. This is the only view where the "void" background disappears entirely. Transition from Standard→Full is a smooth 400ms expansion.

  • Breadcrumb bar: file path segments, clickable

  • Multiple terminal instances with tab switching

Status Bar (bottom, 22px)

  • Left: branch name (clickable for branch switch), sync status

Recommendation

Build C: The Pulse — with Stage's edge glow and Codex's typography confidence.

The orb gives CodeBuddy a face. The per-project color-shifting makes every project YOUR space. The right-side rail becomes unforgettable because the mini orb lives there — always breathing, always alive. And in the IDE tab, the orb sits in the status bar, seamlessly bridging CodeBuddy's identity with the VS Code experience.

From Stage: borrow the edge glow that responds to AI activity.
From Codex: borrow the typography hierarchy — bold Space Grotesk headers, generous spacing.
From Pulse: everything else — the orb, the color DNA, the asymmetric glow, the identity.
The CodeBuddy Identity: A monolith panel in a dark void. A right-side rail with a living orb. The orb shifts color per project. Its glow bleeds asymmetrically from the corner. In the IDE, the orb sits in the status bar. You'd recognize this app from across the room.
